## 第一节：架构决策记录（ADR-CL-001 v3.0）

### 背景与演进历史

#### v1.0：管道内单 Chunk 绑定（已废弃）

CausalLinker 最初被绑定在 `_persist_l2_pipeline` 的局部单 chunk 事务内部。

**废弃原因**：如果在不同时间窗、不同 Chunk 里分别提取出 `Incident` 和 `Decision`，因为 L2 流水线只盯着当前 Chunk，跨 Chunk 的因果边永远无法建立——"单视角断链漏洞"无法修复。

#### v2.0：N×M 增量扫描（已废弃）

为了修复跨 Chunk 断链，设计了独立扫描器 `CausalLinkerScanner`，使用 linker_queue 脏页表 + 原子出队 + 时间窗闭包 + 向量相似度门控。

**废弃根因（经 G博士质检，共四个致命漏洞）**：

| 漏洞编号 | 问题描述 | 性质 |
|----------|----------|------|
| 漏洞9 | 纯类型匹配产生笛卡尔积爆炸：同类型事件 N 条全部互连，O(N²) 边数爆炸 | 致命 |
| 漏洞11 | cos_sim ≠ causality：向量相似度高只说明语义相似，不代表因果关系 | 致命 |
| 漏洞12 | 外键声明可绕过 ontology 校验：`related_event_ids` 直接写入 GraphStore 而不经过 `validate_relation()` | 致命 |
| 漏洞13 | Embedding 粒度不匹配：向量在 chunk 层，event 层向量不存在，gate 无法运行 | 致命 |

向量相似度门控作为 v2.0 的最后防线，被漏洞11、13 彻底否定。四个漏洞组合，v2.0 方案完全失效。

#### v3.0（当前）：外键校验建边

**决策原则**：在 V6.3 offline-first 约束下，不引入 Causal NLI，不做自动因果推断。建边能力**完全依赖 L2 大模型在提取时的显式外键声明**。

### 当前核心原则

1. **L2 显式外键，不得幻构**：L2 大模型仅在原文有明确上下文引用时填充 `related_event_ids`，无明确引用返回 `[]`，禁止幻构因果关系。

2. **ontology 校验强制**：所有外键路径必须经过 `OntologyEngine.validate_relation()` 校验，无论来源是 L2 提取还是手动输入。

3. **孤岛容忍**：无外键事件作为孤岛存在，不再主动寻找前世今生。建边是 L2 的职责，不是扫描器的职责。

4. **同步执行，无扫描器**：`_link_causal_edges` 在事件入库后同步调用，无异步扫描器，无 linker_queue，无 linker_scanned 标志位。

## 第三节：核心接口契约（v3.0）

### _link_causal_edges（主入口）

```python
def _link_causal_edges(self, event: Event) -> None:
    """
    事件入库后同步执行。无扫描器，无时间窗，无队列。

    调用时机：EventIndex.add() 完成后，由 ExtractionPipeline 调用。
    容错：目标事件不存在、关系无法推断、ontology 校验失败 → logger.warning + 跳过，不 raise。
    """
    # P博士裁定：字段名统一为 related_event_ids。
    # EventIndex.get() 通过 Pydantic model_validate 反序列化，already 是 Python list。
    # isinstance 兜底处理直接从 DB row 构建的边缘路径。
    related_ids = event.related_event_ids or []
    if isinstance(related_ids, str):
        related_ids = json.loads(related_ids)
    if not related_ids:
        return

    for target_id in related_ids:
        target = self.event_index.get(target_id)
        if target is None:
            logger.warning(f"[CausalLinker] 目标事件不存在: {target_id}，跳过")
            continue

        # 修正2（漏洞2）：本体关系有方向性（domain → range）。
        # 声明引用的 event 未必是 source，需根据本体规则动态决定方向。
        # 例：Decision 引用 Incident，本体规则是 triggers: Incident → Decision，
        # 必须翻转为 source=Incident, target=Decision，否则 ontology 校验失败。
        result = _infer_directed_relation(event, target)
        if result is None:
            logger.warning(
                f"[CausalLinker] 本体中无合法关系: {event.type} ↔ {target.type}，跳过"
            )
            continue

        relation, source_evt, target_evt = result
        valid, msg = self.ontology.validate_relation(
            relation, source_evt.type, target_evt.type
        )
        if not valid:
            logger.warning(f"[CausalLinker] 本体校验失败: {msg}，跳过")
            continue

        self.graph_store.add_relation(
            source=source_evt.id,
            target=target_evt.id,
            relation=relation,
            weight=1.0,
            metadata={"derived_from_chunks": [event.source_chunk_id]},
        )
```

### 依赖关系

| 依赖 | 类型 | 作用 |
|------|------|------|
| `self.event_index` | `EventIndex` | 读取目标事件（`get(target_id)`） |
| `self.ontology` | `OntologyEngine` | 校验关系合法性（`validate_relation()`） |
| `self.graph_store` | `GraphStore` | 写入因果边（`add_relation()`，内部使用 RMW + BEGIN IMMEDIATE） |

### _infer_directed_relation（辅助函数）

```python
def _infer_directed_relation(
    event_a: Event, event_b: Event
) -> tuple[str, Event, Event] | None:
    """
    在本体中查找 event_a 和 event_b 之间的合法有向关系。
    返回 (relation_name, true_source, true_target)，或 None（无合法关系）。

    实现逻辑：
    1. 先查 domain=event_a.type, range=event_b.type 的关系 → 返回 (relation, event_a, event_b)
    2. 再查 domain=event_b.type, range=event_a.type 的关系 → 返回 (relation, event_b, event_a)（翻转）
    3. 两者都没有 → 返回 None

    注：利用 OntologyEngine.list_valid_relations(source_class) 查询，
    无需新增 OntologyEngine 接口。
    """
    ontology = get_ontology()
    # 正向：event_a 为 source
    for rel in ontology.list_valid_relations(event_a.type):
        rel_def = ontology.get_relation(rel)
        if rel_def and rel_def.range == event_b.type:
            return (rel, event_a, event_b)
    # 反向：event_b 为 source（关系方向与声明引用方向相反）
    for rel in ontology.list_valid_relations(event_b.type):
        rel_def = ontology.get_relation(rel)
        if rel_def and rel_def.range == event_a.type:
            return (rel, event_b, event_a)
    return None
```

> **字段名（已裁定）**：统一使用 `related_event_ids`（P博士裁定，2026-02-23）。
> `models.py` 字段已同步改名；DB 列名、migration 002、三份文档代码契约全部对齐。
