## 问题背景

`dimc down` 将原始 Claude JSONL 转录复制进 `end.md`，但没有提取结构化事件。
`run_smart_scan()` 完全依赖 `DEEPSEEK_API_KEY`，本地无 API Key 的环境零提取。

**目标**：三层提取流水线 — L1（BGE-M3，始终离线）提供基线提取；L2（云端 LLM，可选）精炼；CausalLinker 退化为事件入库时的同步外键校验（无自动推断，无 N×M 扫描）。

## 定案总数：26 条（全部封闭）

### 定案 1-25：提取流水线 + Option C

| # | 定案 |
|---|------|
| 1 | `chunk_id = "chk_" + sha256(f"{source_event_id}:{content[:64]}").hexdigest()[:16]` |
| 2 | chunks.db **独立**（`~/.dimcause/chunks.db`），不放入 index.db / code.db |
| 3 | `session_id` 是 chunks 表一级字段 |
| 4 | `event_ids` 是 JSON 数组字段（一个 chunk 可对应多个事件） |
| 5 | L1/L2 **命名空间隔离**：L1→`sha256(chunk_id:l1:index)`，L2→`sha256(chunk_id:l2:index)` |
| 6 | `threading.local()` TLS 连接池 + `PRAGMA busy_timeout=5000` |
| 7 | 两个独立整数版本字段：`embedding_version` / `extraction_version` |
| 8 | **`superseded_by` 撤出**：Option C 完全取代 M-4，events 表不加此字段 |
| 9 | events 表新增 `source_chunk_id TEXT DEFAULT NULL` |
| 10 | events 表新增 `source_layer TEXT DEFAULT NULL CHECK(source_layer IS NULL OR source_layer IN ('l1', 'l2'))` |
| 11 | L1 写入策略：`INSERT OR IGNORE`（同 chunk 重跑不重复） |
| 12 | L2 写入策略：组级 DELETE + INSERT（单库事务），彻底清除同 chunk 旧 L2 行 |
| 13 | CausalLinker 代表事件：**per-chunk per-type**（每个 chunk 每种 event_type 取一个，L2 > L1） |
| 14 | 查询 COALESCE：UNION ALL 两路（路径 A 流水线事件，路径 B 历史事件） |
| 15 | 跨库写入顺序：**event_index.db 先，chunks.db 后** |
| 16 | `event_ids` 更新封装在 ChunkStore 内（不暴露给调用方） |
| 17 | L1 GC：延迟到 V6.4 |
| 18 | COALESCE 过滤机制：`source_layer` + `EXISTS / NOT EXISTS` |
| 19 | `source_chunk_id IS NULL` 行（历史事件）走路径 B，全量返回 |
| 20 | CausalLinker 过滤粒度：per-chunk **per-type** |
| 21 | 索引：`idx_chunk_layer ON events(source_chunk_id, source_layer, updated_at DESC)` |
| 22 | CHECK CONSTRAINT NULL 安全模式：`IS NULL OR ... IN (...)` |
| 23 | 上述 NULL 规则适用于所有有 NULL 允许值的枚举列 |
| 24 | graph_edges 字段名：`source` / `target` / `relation`（非 source_id/target_id） |
| 25 | `add_relation()` 策略：**RMW + BEGIN IMMEDIATE**（weight 取 MAX，metadata 列表去重追加，`created_at` 首次写入后不变） |

### 定案 26：CausalLinker 外键校验建边（N×M 扫描系统全部废弃）

> **废弃内容**（经13轮审查后裁定不可实现）：
> - `linker_queue` 表、`linker_scanned` 列、偏索引 → 不建
> - `linker_scanner.py`（增量扫描器）→ 不建
> - `migrations/003_linker_queue.py` → 不建
> - N×M 时窗闭包、原子出队、视野池、`_filter_ghost_edges`、`scan_once()` → 全部删除
> - DEV_ONTOLOGY_RULES 纯类型匹配建边（笛卡尔积爆炸）→ 废弃
> - 向量相似度门控（cos_sim）→ 废弃（语义相似 ≠ 因果，粒度错位）
>
> **废弃根因**：V6.3 offline-first 约束下，无 LLM 推断、无标注数据，任何自动因果推断机制均不可靠。

**定案 26 正文**：

- events 表新增 `related_event_ids TEXT DEFAULT '[]'`，L2 提取时填充，L1 阶段留空 `[]`
- L2 大模型在原文本有明确上下文引用时，在 payload 中返回 `related_event_ids: ["evt_xxx"]`；若无明确关联，返回空数组，**不得幻构**
- 事件入库（`EventIndex.add()`）后，遍历 `related_event_ids`：
  - 查询目标事件 type
  - 调用 `OntologyEngine.validate_relation(relation, source.type, target.type)`
  - 校验通过 → `GraphStore.add_relation()`（RMW，定案 25）
  - 校验失败或目标事件不存在 → WARNING 日志，跳过，不写边
- 无 `related_event_ids` 的事件作为**孤岛**存在，不寻找前世今生
- `_get_conn()` 统一 `isolation_level=None` + `PRAGMA foreign_keys = ON`（支持 add_relation RMW）
- 迁移守卫统一写法：`cols = {row[1] for row in conn.execute("PRAGMA table_info(events)")}` 而非 SQL IF NOT EXISTS

## 架构图

```
session_end.py
  └── ExtractionPipeline.run(session_id)
        ├── L0 Ingest:   raw text → chunks.db (status='raw')
        ├── L1 Embed:    BGE-M3 → event_vectors (status='embedded', confidence='low')
        ├── L1 Extract:  SessionExtractor → events(source_layer='l1') INSERT OR IGNORE
        ├── L2 Extract:  [optional] LLM → DELETE旧L2 + INSERT新L2(source_layer='l2')
        │              → payload 含 related_event_ids（L2 在原文有明确引用时填充）
        ├── Validate:    Pydantic ExtractedEventRecord 门
        ├── Persist:     event_index.db 先写，chunks.db event_ids 后更新
        └── CausalLinker（同步外键校验，无扫描器）：
              ├── 遍历 event.related_event_ids
              ├── OntologyEngine.validate_relation() 校验
              ├── 通过 → GraphStore.add_relation() RMW
              └── 不通过 / 无外键 → WARNING 日志，事件作为孤岛存在
```

**FSM（chunks.status）**:
```
  raw ──embed──► embedded ──extract──► extracted
         │                    │
      [fail]               [fail]
      retry_count++        retry_count++
      if >= MAX_RETRIES:   if >= MAX_RETRIES:
        extraction_failed=True, needs_extraction=False
```

## DDL

### chunks.db（独立数据库）

```sql
PRAGMA journal_mode = WAL;
PRAGMA synchronous = NORMAL;
PRAGMA busy_timeout = 5000;

CREATE TABLE IF NOT EXISTS chunks (
    chunk_id           TEXT PRIMARY KEY,
    source_event_id    TEXT NOT NULL,
    session_id         TEXT NOT NULL,
    content            TEXT,
    status             TEXT NOT NULL DEFAULT 'raw'
                           CHECK(status IN ('raw', 'embedded', 'extracted')),
    confidence         TEXT DEFAULT 'low'
                           CHECK(confidence IS NULL OR confidence IN ('low', 'high')),
    needs_extraction   BOOLEAN DEFAULT 1,
    embedding_version  INTEGER DEFAULT 0,
    extraction_version INTEGER DEFAULT 0,
    retry_count        INTEGER DEFAULT 0,
    extraction_failed  BOOLEAN DEFAULT 0,
    event_ids          TEXT DEFAULT '[]',
    last_error         TEXT,
    created_at         DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at         DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX IF NOT EXISTS idx_chunks_session ON chunks(session_id);
CREATE INDEX IF NOT EXISTS idx_chunks_status  ON chunks(status);
CREATE INDEX IF NOT EXISTS idx_chunks_needs   ON chunks(needs_extraction)
    WHERE needs_extraction = 1;
```

## 核心 SQL / 实现逻辑

### COALESCE 查询（UNION ALL 两路，定案 14、18、19）

```sql
-- 路径 A：流水线事件（有 source_chunk_id），L2 优先覆盖 L1
SELECT e.* FROM events e
WHERE e.source_chunk_id IS NOT NULL
  AND (
    (e.source_layer = 'l2' AND EXISTS (
        SELECT 1 FROM events e2
        WHERE e2.source_chunk_id = e.source_chunk_id AND e2.source_layer = 'l2'))
    OR
    (e.source_layer = 'l1' AND NOT EXISTS (
        SELECT 1 FROM events e2
        WHERE e2.source_chunk_id = e.source_chunk_id AND e2.source_layer = 'l2'))
  )
UNION ALL
-- 路径 B：历史事件（无 source_chunk_id），全量返回
SELECT e.* FROM events e
WHERE e.source_chunk_id IS NULL;
```

### L2 组级清洗写入（定案 12）

```sql
-- 单库事务内
DELETE FROM events WHERE source_chunk_id = ? AND source_layer = 'l2';
INSERT INTO events (..., source_chunk_id, source_layer, related_event_ids)
    VALUES (..., ?, 'l2', ?);
```

### CausalLinker 外键校验建边（定案 26）

```python
def _link_causal_edges(self, event: Event) -> None:
    """事件入库后同步执行，无扫描器，无时间窗"""
    # P博士裁定：字段名统一为 related_event_ids。
    # EventIndex.get() 通过 Pydantic model_validate 反序列化，already 是 Python list。
    related_ids = event.related_event_ids or []
    if isinstance(related_ids, str):
        related_ids = json.loads(related_ids)
    if not related_ids:
        return
    for target_id in related_ids:
        target = self.event_index.get(target_id)
        if target is None:
            logger.warning(f"[CausalLinker] 目标事件不存在: {target_id}，跳过")
            continue
        # 双向搜索本体，动态决定 source/target 方向，防止拓扑倒置。
        result = _infer_directed_relation(event, target)
        if result is None:
            logger.warning(
                f"[CausalLinker] 本体中无合法关系: {event.type} ↔ {target.type}，跳过"
            )
            continue
        relation, source_evt, target_evt = result
        valid, msg = self.ontology.validate_relation(
            relation, source_evt.type, target_evt.type
        )
        if not valid:
            logger.warning(f"[CausalLinker] 本体校验失败: {msg}，跳过")
            continue
        self.graph_store.add_relation(
            source=source_evt.id,
            target=target_evt.id,
            relation=relation,
            weight=1.0,
            metadata={"derived_from_chunks": [event.source_chunk_id]},
        )
```

### add_relation() RMW（定案 25）

```python
def add_relation(self, source, target, relation, weight=1.0, metadata=None):
    metadata = metadata or {}
    conn = self._get_conn()           # isolation_level=None（定案 26）
    conn.execute("BEGIN IMMEDIATE")
    try:
        row = conn.execute(
            "SELECT weight, metadata FROM graph_edges "
            "WHERE source=? AND target=? AND relation=?",
            (source, target, relation)
        ).fetchone()
        if row:
            new_weight = max(row[0], weight)
            new_meta = _merge_metadata(json.loads(row[1] or '{}'), metadata)
            conn.execute(
                "UPDATE graph_edges SET weight=?, metadata=? "
                "WHERE source=? AND target=? AND relation=?",
                (new_weight, json.dumps(new_meta), source, target, relation)
            )
        else:
            conn.execute(
                "INSERT INTO graph_edges (source, target, relation, weight, metadata, created_at) "
                "VALUES (?, ?, ?, ?, ?, ?)",
                (source, target, relation, weight, json.dumps(metadata), time.time())
            )
        conn.execute("COMMIT")
    except Exception:
        conn.execute("ROLLBACK")
        raise
    finally:
        conn.close()
```

### _get_conn() 统一规范（定案 26）

```python
def _get_conn(self) -> sqlite3.Connection:
    conn = sqlite3.connect(self.db_path, isolation_level=None)
    conn.execute("PRAGMA foreign_keys = ON")
    conn.execute("PRAGMA journal_mode = WAL")
    conn.row_factory = sqlite3.Row
    return conn
```
