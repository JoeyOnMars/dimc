# 研究文档

**状态**：当前有效
**定位**：收纳仍有参考价值的技术调研报告，不承担待办追踪、分支模板或阶段计划。
**不包含**：研究待办清单、占位课题、未落地模板、过时的流程脚本。

## 1. 当前保留文件

1. [RT-000_model_selection_evaluation.md](./RT-000_model_selection_evaluation.md)
   - 当前有效的模型选型基线。
   - 仍被 live 文档和代码注释引用。

2. [RT-001_ai_tools_integration.md](./RT-001_ai_tools_integration.md)
   - 2026-02 的工具集成调研快照。
   - 保留为历史研究材料，不作为当前接线指南。

## 2. 当前入口规则

1. `docs/research/` 只保留真正有参考价值的研究报告。
2. 不再单独维护 `index.md` 这类研究待办追踪页。
3. 研究是否进入共享当前面，以“是否仍有参考价值”判断，不以“曾经创建过课题”判断。

## 3. 与其他文档面的边界

1. `docs/research/`
   - 负责技术调研、方案评估、选型背景。

2. `docs/dev/`
   - 负责路线图、技术债、整改计划、阶段实施清单。

3. `docs/guides/`
   - 负责当前可执行的操作指南。

## 4. 维护规则

1. 若新增研究文稿，直接新增 `RT-XXX_*.md` 文件，并在本文登记。
2. 若研究结论已经失效、且不再被 live 文档或代码引用，应删除或迁出共享当前面。
3. 若研究只是历史快照，应在文稿开头明确写出“历史研究报告”，避免被误当成 live 指南。
