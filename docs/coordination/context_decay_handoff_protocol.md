# 上下文压缩与线程接力协议

**状态**：当前有效  
**定位**：仓库治理层中的协作连续性协议。  
**边界**：本文只定义“如何在长会话中防止上下文断裂”，不定义产品能力。

## 1. 触发条件

满足任一条件即触发接力动作：

1. 对话出现“上下文压缩”提示或明显记忆衰减。
2. 单线程连续推进超过 45 分钟且任务跨越多个子目标。
3. 准备切线程、切分支、切工作线或跨人/跨 agent 交接。

## 2. 双层载体

1. **共享协议层**（可复用）：本文件。
2. **本地现场层**（实时快照）：`tmp/discussion/SESSION_HANDOFF_LIVE.md`。

约束：

1. 共享层只写方法，不写一次性现场细节。
2. 本地层只写事实和下一步，不写长篇背景叙述。

## 3. 本地快照最小字段

`SESSION_HANDOFF_LIVE.md` 每次更新必须至少包含：

1. 时间戳（本地时区）与当前 `HEAD`。
2. 本轮已完成（只写事实）。
3. 当前进行中（只保留 1 条主线）。
4. 下一步三件事（按执行顺序）。
5. 已验证命令与结果摘要。
6. 阻断/风险与处理策略。

## 4. 更新频率

1. 长任务每 30-45 分钟更新一次本地快照。
2. 完成关键里程碑后立即更新一次。
3. 切线程前必须更新一次，作为唯一接力入口。

## 5. 恢复流程（新线程）

新线程恢复时按以下顺序：

1. 读取 `tmp/discussion/SESSION_HANDOFF_LIVE.md`。
2. 执行 `git status --short --branch` 与 `git log -1 --oneline` 复核基线。
3. 仅按“下一步三件事”执行，不重做已完成项。
4. 若基线不一致，先写差异，再重排下一步。

## 6. 验收标准

1. 新线程在 3 分钟内能恢复“当前状态 + 下一步”。
2. 同一问题不因上下文压缩反复返工。
3. 状态汇报可由快照文件直接复述，不依赖长聊天记录。
