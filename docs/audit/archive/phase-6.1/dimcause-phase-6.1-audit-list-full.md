# 🎯 审计执行指南

## 步骤1：逐项检查

对照V2计划，逐项评分：
- 引用具体代码位置（如"脚本L40"）
- 关联相关章节（如"Sec 5.1"）
- 标注User Correction编号（如适用）

## 步骤2：记录缺失项

明确列出所有❌和⚠️项，并标注：
- 是否阻塞执行（P0/P1/P2）
- 修复成本估算
- 建议处理方式

## 步骤3：计算最终得分

示例：
- 通过：70项 → 70×1.0 = 70
- 部分通过：15项 → 15×0.5 = 7.5
- 失败：5项 → 5×0 = 0
- **总得分**：(70+7.5)/90×10 = **8.6/10** ✅

## 步骤4：撰写审计报告

包含：
- 总得分与评级
- 核心亮点（各类别通过率）
- 缺失项清单+阻塞级别
- 执行建议（批准/修正）
