# 审计归档说明

## 定位

1. 本目录保存历史审计、阶段计划、AI 会诊稿、旧分析和原始 trace。
2. 本目录默认不作为当前事实依据。
3. 这里的内容保留，是为了追溯项目演化和错误来源，不是为了充当正式入口。

## 子目录

1. `analysis/`
   - 历史分析与阶段性判断。
   - 可能包含已过时结论。
2. `ai-generated/`
   - 外部 AI 或多轮 AI 会诊产物。
   - 默认视为讨论材料，不视为正式审计。
3. `historical-audits/`
   - 旧审计和修复报告。
   - 仅保留追溯价值。
4. `phase-6.1/`
   - 某一阶段的计划、验证证明、清单与原始输出。

## 使用边界

1. 如果某项结论已经被正式文档或 live 代码覆盖，应以新的事实为准。
2. 如果要引用归档内容，必须显式说明它是“历史快照”。
3. 不得把 `archive/` 下文档当成当前产品状态、当前治理状态或当前实现状态的直接依据。
