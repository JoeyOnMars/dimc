### 🚀 V6.1 - V6.2 执行优先级路线 (Execution Plan)

> 基于架构审阅重塑的三阶段计划，按严格的依赖与质量红线顺序执行。

**第一阶段：稳固地基与质量防线 (Priority: P0-P1)**
1. **任务 3.1 & 3.2**: 全量测试红线清理 (消除 legacy_debt) + Pyre2 静态分析配置修复 (消除虚假报错)。
    *说明：测试与类型检查是所有后续重构的安全网，必须最优先执行。*
2. **信任梯度 (Risk Level)**: 契约模板引入 risk_level 分级，固化审批流程基础设施。
3. **任务 1.7**: MCP Server 配置修复，保障外部 AI 链路。

**第二阶段：本体知识层补齐 (Priority: P0)**
4. **L3 本体防波堤补齐 (SchemaValidator)**: 落实运行时校验，拦截非法实体/关系，保障推理引擎数据质量。

**第三阶段：基础设施真实化与发布 (Priority: P1-P2)**
5. **L0 调度底座补齐 (Orchestrator)**: 从骨架演进为实际的后台任务循环与进程常驻。
6. **L1 自动化接入 (dimc detect)**: 实现 IDE 环境自动探测。
7. **任务 1.9**: 文档对齐 (Docs Alignment)，确保 README 与内部架构文档一致。
8. **Release 准备**: PyPI 发布 + .gitignore 清理。

*(L4 推理引擎补齐作为探索性任务，置于上述核心依赖完成后)*
